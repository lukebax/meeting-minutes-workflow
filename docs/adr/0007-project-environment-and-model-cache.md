# Project Environment and Model Cache

The workflow will keep project Python dependencies in a repo-local `.venv`, while large local transcription model files will live in the transcription tool's normal user-level cache rather than in the repository or virtual environment. Codex-assisted setup may download the required transcription model after making that visible to the user, so setup remains simple without making the Git repository large or project-specific model copies. System-level tools such as audio decoders are detected and explained separately from project-local dependencies, with installation requiring explicit user approval. Setup checks should report readiness separately for transcript workflows, audio transcription, and Word export, so transcript inputs can still work when audio dependencies are missing.
