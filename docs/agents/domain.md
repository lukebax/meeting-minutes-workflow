# Domain Docs

This repository uses a single-context domain documentation layout.

Read these files before architectural, diagnostic, TDD, triage, PRD, or issue-breakdown work:

- `CONTEXT.md` for project language
- `docs/adr/` for architectural decisions
- `docs/PROJECT_PLAN.md` for implementation sequencing
- `docs/IMPLEMENTATION_RESEARCH.md` for researched tool choices and open technical questions
- `HOW_TO_USE.md` for the intended non-technical user flow

Keep `CONTEXT.md` as a glossary only. Put setup instructions, implementation plans, checklists, and specs in the appropriate docs file instead.
